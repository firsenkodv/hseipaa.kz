MoonShine v4 — Отдельные страницы с сохранением данных

Как создать независимую страницу в панели администратора (не ресурс), добавить на неё поля и сохранять данные в базу данных через универсальную таблицу настроек.

Концепция

В MoonShine v4 существует два типа объектов меню:

  • ModelResource — CRUD для набора записей (таблица со списком, форма редактирования).
  • Page — произвольная страница с любыми компонентами. Никакой таблицы «по умолчанию» у неё нет.

Для хранения данных настроечных страниц (Главная, Контакты, О нас и т.д.) не нужно создавать отдельную таблицу под каждую страницу. Достаточно одной универсальной таблицы settings, где каждая страница хранит свои данные в одной строке в формате JSON, идентифицируя строку по полю group.

idgroupdata (JSON)
1home{"hero_title": "Добро пожаловать", "hero_subtitle": "..."}
2contacts{"phone": "+7 (727) 224 21 21", "email": "..."}
3about{"text": "О компании...", "img": "..."}

Строка создаётся автоматически при первом открытии страницы — никакие дополнительные миграции или модели для новых страниц не нужны.

Пошаговое создание

1

Миграция универсальной таблицы settings

Создаётся один раз для всего проекта. Новые страницы не требуют новых миграций.

// database/migrations/…_create_settings_table.php

Schema::create('settings', function (Blueprint $table): void {
    $table->id();
    $table->string('group')->unique();  // идентификатор страницы
    $table->json('data')->nullable();    // все поля страницы в JSON
    $table->timestamps();
});
2

Модель Setting

Поле data кастуется как массив. Метод getGroup() возвращает строку для нужной страницы, создавая её автоматически если она ещё не существует. 

// app/Models/Setting.php

class Setting extends Model
{
    protected $fillable = ['group', 'data'];

    protected $casts = [
        'data' => 'array',
    ];

    public static function getGroup(string $group): self
    {
        // Найти строку по group или создать новую с пустым data
        return static::firstOrCreate(
            ['group' => $group],
            ['data'  => []]
        );
    }
}
3

Класс страницы

Страница расширяет MoonShine\Laravel\Pages\Page. Форма строится через FormBuilder, заполняется методом fill() из массива data нужной группы настроек. Сохранение происходит через #[AsyncMethod] — без отдельного контроллера. 

// app/MoonShine/Pages/Pages/HomePage.php

class HomePage extends Page
{
    public function getTitle(): string
    {
        return 'Главная';
    }

    // Получить (или создать) строку настроек для этой страницы
    private function getSetting(): Setting
    {
        return Setting::getGroup('home');
    }

    // Асинхронный обработчик сохранения формы
    #[AsyncMethod]
    public function store(Request $request): JsonResponse
    {
        $setting = $this->getSetting();
        $setting->data = $request->except(['_token', '_method']);
        $setting->save();

        return JsonResponse::make()->toast('Сохранено', ToastType::SUCCESS);
    }

    // Построить форму
    private function form(): FormBuilder
    {
        return FormBuilder::make()
            ->asyncMethod('store')           // POST на #[AsyncMethod] store()
            ->fill($this->getSetting()->data ?? []) // заполнить из JSON
            ->fields([
                Box::make('Главный баннер', [
                    Text::make('Заголовок',    'hero_title'),
                    Text::make('Подзаголовок', 'hero_subtitle'),
                    TinyMce::make('Описание',    'hero_description'),
                    Image::make('Изображение', 'hero_image')
                        ->disk('public')
                        ->dir('pages/home')
                        ->removable(),
                ]),
                Box::make('Приветственный блок', [
                    TinyMce::make('Текст', 'welcome_text'),
                ]),
            ])
            ->submit('Сохранить', ['class' => 'btn-primary']);
    }

    // Отрендерить страницу
    protected function components(): iterable
    {
        yield $this->form();
    }
}
Ключевой момент: fill(array) заполняет форму из простого массива. Имена полей (hero_title, hero_subtitle и т.д.) напрямую соответствуют ключам в JSON-колонке data. Никакой дополнительной типизации не нужно.
4

Регистрация страницы в ServiceProvider

// app/Providers/MoonShineServiceProvider.php

$core
    ->resources([...])
    ->pages([
        ...$core->getConfig()->getPages(), // системные страницы MoonShine
        HomePage::class,               // наша страница
    ]);

Как добавить новую страницу (например, «Контакты»)

Новая страница не требует новой таблицы или миграции. Достаточно трёх действий:

  1. Создать файл app/MoonShine/Pages/Pages/ContactsPage.php по образцу HomePage, заменив:
    • getTitle() → 'Контакты'
    • Setting::getGroup('home') → Setting::getGroup('contacts')
    • поля формы на нужные (телефон, email, адрес и т.д.)
  2. Зарегистрировать ContactsPage::class в MoonShineServiceProvider.
  3. Добавить MenuItem::make(ContactsPage::class, 'Контакты', 'phone') в меню.
Строка в таблице settings с group = 'contacts' будет создана автоматически при первом сохранении формы через firstOrCreate().

Чтение данных на сайте (во views)

Данные страницы доступны из любого шаблона или контроллера:

// В контроллере
$home = Setting::getGroup('home');
$heroTitle = $home->getValue('hero_title', 'Заголовок по умолчанию');

// Или напрямую
$data = Setting::getGroup('home')->data;
$heroTitle = $data['hero_title'] ?? '';
{{-- В Blade-шаблоне --}}
@php($home = \App\Models\Setting::getGroup('home'))

<h1>{{ $home->getValue('hero_title') }}</h1>
<p>{!! $home->getValue('hero_description') !!}</p>
Важно: используй {!! !!} (без экранирования) для полей TinyMce, так как они содержат HTML. Для обычных текстовых полей используй {{ }}.
